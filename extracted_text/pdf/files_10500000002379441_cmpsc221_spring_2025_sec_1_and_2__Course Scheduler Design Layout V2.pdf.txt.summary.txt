The Course Scheduler design uses `Entry` classes (e.g., `CourseEntry`) to model database table rows/columns, with primary keys marked `*`. `Query` classes (e.g., `CourseQueries`) handle database access/updates, returning `ArrayLists` of `Entry` objects. This establishes a 3-tier application structure: GUI, class code, and database tables (Semester, Course, Class, Student, Schedule).
